# AI Code Helper - 开发文档

> 版本：1.0.0 | 更新日期：2026-03-08

---

## 项目概述

AI Code Helper 是一个全栈 AI 编程助手，由 Vue 3 前端 + Spring Boot 4 后端组成，  
集成阿里云 DashScope（通义千问）大模型，具备以下核心能力：

- **SSE 流式对话**：逐 token 实时推送，打字机效果
- **多轮对话记忆**：基于 `memoryId` 的多会话隔离
- **RAG 检索增强**：本地文档向量化，自动注入上下文
- **网络实时搜索**：大模型按需调用 WebSearchTool
- **输入安全防护**：SafeInputGuardrail 前置过滤危险请求

## 模块说明

### 后端模块

#### AiCodeHelperApplication.java

Spring Boot 启动类，同时负责注册全局 CORS 配置 Bean。  
允许所有来源、方法、请求头访问，并支持携带凭证（Cookie）。

> 生产环境应将 `allowedOriginPatterns("*")` 替换为具体域名。

#### AiCodeHelperService.java

AI 服务核心接口（由 LangChain4j `@AiService` 代理实现）：

| 方法 | 返回类型 | 说明 |
|------|---------|------|
| `chat(String)` | `String` | 同步对话 |
| `chatReport(String)` | `Report` | 结构化报告 |
| `chatWithRag(String)` | `Result<String>` | 带 RAG 置信度的对话 |
| `chatStream(int, String)` | `Flux<String>` | SSE 流式对话（核心接口） |

接口上标注了 `@InputGuardrails({SafeInputGuardrail.class})`，所有方法调用前均会经过安全校验。

系统提示词通过 `@SystemMessage(fromResource = "system-prompt.txt")` 从资源文件加载。

#### AiCodeHelperServiceFactory.java

Spring `@Configuration` 工厂类，使用 `AiServices.builder()` 组装 AI 服务：

- 注册流式模型（`streamingChatModel`）用于 `chatStream`
- 注册同步模型（`chatModel`）用于其他方法
- 注册 `PersistentChatMemoryStore` 实现多会话记忆
- 注册 `ContentRetriever` 实现 RAG 自动检索
- 注册 `WebSearchTool` 供模型按需调用

#### PersistentChatMemoryStore.java

实现 `ChatMemoryStore` 接口，使用 `ConcurrentHashMap<Object, List<ChatMessage>>` 存储每个 `memoryId` 的对话历史。

> 当前为内存存储，应用重启后历史丢失。生产环境可替换为 Redis 实现。

#### AiController.java

RESTful 控制器，提供 `GET /ai/chat` SSE 端点。  
接收 `memoryId` 和 `message` 参数，调用 `aiCodeHelperService.chatStream()`，  
将 `Flux<String>` 中每个 token 包装为 `ServerSentEvent<String>` 推送给前端。

#### RagConfig.java

RAG 检索增强配置类：

1. 从 `src/main/resources/docs/` 加载所有文档（支持 .md、.txt 等格式）
2. 使用 `DocumentByParagraphSplitter` 按段落切分（每段最大 1000 字符，重叠 200 字符）
3. 通过 `text-embedding-v3` 生成向量，存入 `EmbeddingStore`（内存向量库）
4. 构建 `EmbeddingStoreContentRetriever`，每次检索返回相似度 ≥ 0.75 的最多 5 条结果

#### SafeInputGuardrail.java

LangChain4j `InputGuardrail` 实现，在用户消息到达模型前进行安全校验。  
当前拦截「删除」系列关键词，命中时返回 `fatal` 结果终止调用链。

#### WebSearchTool.java

LangChain4j `@Tool` 工具，供大模型按需调用：

- 通过 Jsoup 爬取百度搜索结果页
- 提取 `h3` 标题和摘要，最多返回 `maxResults` 条
- 支持通过 `application.yml` 配置 `userAgent`、`timeoutMillis`、`maxResults`

#### AiCodeHelper.java

基础 AI 服务类（独立于 LangChain4j Service 体系）：

- `chat(String)`: 直接调用 `ChatModel` 进行同步对话
- `chatM()`: 使用 DashScope SDK 原生 API 测试连通性

### 前端模块

#### App.vue

主聊天界面，包含：

- **消息列表**：渲染用户/AI 消息，支持代码块高亮（轻量处理）
- **输入框**：`Enter` 发送，`Shift+Enter` 换行
- **新建对话**：清空消息列表并递增 `memoryId`
- **流式追加**：`onMessage` 回调中逐步追加 AI 回复内容
- **自动滚屏**：每次新内容追加后滚动到底部

#### api/chat.js

封装 `EventSource` 的两个函数：

- `chatStream(memoryId, message, callbacks)`: 流式请求，立即返回取消函数
- `chat(memoryId, message)`: Promise 包装，等待全部 token 后 resolve

## 开发环境搭建

### 后端

```bash
# 1. 确认 JDK 版本
java -version   # 需要 21+

# 2. 配置 API Key（勿提交到 Git）
# 编辑 src/main/resources/application-local.yml

# 3. 启动
./mvnw spring-boot:run

# 或 IDE 中直接运行 AiCodeHelperApplication.main()
```

### 前端

```bash
cd front
pnpm install
pnpm dev
# 访问 http://localhost:5173
```

## 关键配置项

| 配置路径 | 默认值 | 说明 |
|---------|--------|------|
| `server.port` | 8080 | 后端端口 |
| `server.servlet.context-path` | `/api` | 接口路径前缀 |

## 扩展开发指南

### 新增 Tool（工具）

1. 创建新类，添加 `@Tool` 注解的方法：
```java
@Component
public class MyTool {
    @Tool(name = "myTool", value = "工具功能描述")
    public String doSomething(@P("参数说明") String input) {
        // 实现逻辑
        return result;
    }
}
```

2. 在 `AiCodeHelperServiceFactory.getAiCodeHelperService()` 中注册：
```java
.tools(new WebSearchTool(), new MyTool())
```

### 新增 RAG 文档

将 `.md` 或 `.txt` 文件放入 `src/main/resources/docs/`，重启后端即可自动向量化索引。

### 扩展安全校验

在 `SafeInputGuardrail.validate()` 中添加更多规则，例如正则匹配：

```java
if (Pattern.compile("rm\s+-rf|DROP\s+TABLE", Pattern.CASE_INSENSITIVE)
        .matcher(s).find()) {
    return fatal("检测到高危操作指令，已拒绝");
}
```

### 持久化会话历史

实现 `ChatMemoryStore` 接口，使用 Redis 替代内存存储：

```java
@Service
public class RedisChatMemoryStore implements ChatMemoryStore {
    // 使用 RedisTemplate 实现 getMessages/updateMessages/deleteMessages

```

## 常见问题

### Q: 启动时报 EmbeddingModel 相关错误

确认 `application-local.yml` 中 `embedding-model.api-key` 已配置，且与 chat-model 的 api-key 一致（同一账号下可共用）。

### Q: RAG 检索不到文档内容

- 检查 `rag.docs-path` 路径是否正确（相对于工作目录）
- 确认文档格式支持（.md、.txt、.pdf 等）
- 调低 `rag.min-score` 阈值（如 0.5）观察是否能命中

### Q: WebSearchTool 搜索结果为空

- 百度搜索页结构可能已更新，检查 CSS 选择器 `div.result` 是否还有效
- 可以更换为其他搜索引擎或使用官方搜索 API

### Q: 前端收不到 SSE 消息

- 确认 Vite proxy 配置指向正确的后端地址
- 检查浏览器开发者工具 Network 标签，确认 `/api/ai/chat` 响应头 `Content-Type: text/event-stream`
- 确认后端 `AiController` 返回 `Flux<ServerSentEvent<String>>`
