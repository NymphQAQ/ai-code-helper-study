# 🖼️ 图片压缩和优化指南

## ⚠️ 重要提示

您的原始照片文件非常大（每张 10-25MB），直接在网页上使用会导致：
- ❌ 首页加载速度极慢（可能需要几分钟）
- ❌ 流量消耗巨大（访问者需要下载数百MB数据）
- ❌ 移动端流量费用高昂
- ❌ 用户体验极差

**必须压缩后再使用！**

---

## 📊 优化目标

### 缩略图要求（首页展示）
- **格式**: WebP (优先) 或 JPEG
- **尺寸**: 长边 1200px（短边按比例）
- **质量**: 75-85%
- **文件大小**: <200KB (目标 100-150KB)
- **数量**: 前 12 张精选作品

### 原图要求（画廊页）
- **格式**: JPEG
- **尺寸**: 长边 2000-2500px（保留细节）
- **质量**: 85-90%
- **文件大小**: <1MB (目标 500-800KB)

---

## 🔧 推荐压缩方案

### 方案一: 在线工具（最简单）⭐ 推荐

#### 1. TinyPNG/TinyJPG (批量压缩)
- 网址: [https://tinypng.com/](https://tinypng.com/)
- 支持: 最多 20 张/次
- 优点: 一键压缩，效果好
- 步骤:
  1. 访问网站
  2. 拖拽图片上传
  3. 等待压缩完成
  4. 批量下载

#### 2. Squoosh (单张优化)
- 网址: [https://squoosh.app/](https://squoosh.app/)
- 优点: 可视化对比，支持 WebP
- 步骤:
  1. 上传图片
  2. 调整质量滑块（建议 75-85%）
  3. 对比原图和压缩效果
  4. 下载优化后的图片

#### 3. CompressJPEG (批量)
- 网址: [https://compressjpeg.com/](https://compressjpeg.com/)
- 支持: 批量压缩
- 优点: 操作简单

- 免费开源

### 方案三: 命令行工具 (适合技术人员)

#### 1. ImageMagick
```bash
# 安装
brew install imagemagick

# 压缩单张
convert DSC05537.JPG -resize 1200x -quality 80 thumbnail1.jpg

# 批量压缩
for file in *.JPG; do
  convert "$file" -resize 1200x -quality 80 "compressed/$file"
done
```

#### 2. cwebp (WebP 转换)
```bash
# 安装
brew install webp

# 转换为 WebP
cwebp -q 80 DSC05537.JPG -o DSC05537.webp

# 批量转换
for file in *.JPG; do
  cwebp -q 80 "$file" -o "${file%.JPG}.webp"
done
```

#### 3. sharp (Node.js)
```bash
npm install -g sharp-cli

# 压缩
sharp -i DSC05537.JPG -o thumbnail1.jpg resize 1200 --quality 80
```

## 📁 文件组织

压缩完成后，将文件按以下结构组织：

```
public/
├── placeholder/           # 保留原图备份
│   └── *.JPG
├── thumbnails/           # 缩略图 (<200KB)
│   ├── photo1.webp
│   ├── photo2.webp
│   └── ...
└── originals/            # 画廊页原图 (<1MB)
    ├── photo1.jpg
    ├── photo2.jpg
    └── ...
```

## 💡 专业建议

### 1. 作品选择
- 选择构图优秀的作品
- 确保光线和色彩表现良好
- 展示多样化的拍摄风格
- 人物表情自然、生动

### 2. 压缩质量平衡
- 缩略图: 75-80% 质量（文件小，加载快）
- 原图: 85-90% 质量（保留细节，文件可控）

### 3. 格式选择
- **WebP**: 体积最小，现代浏览器支持好
- **JPEG**: 兼容性最好，所有浏览器支持
- **推荐**: 缩略图用 WebP，原图用 JPEG

### 4. 渐进式加载
- 确保代码中的 `loading="lazy"` 保留
- 用户滚动到图片位置时才加载
