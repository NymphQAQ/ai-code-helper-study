# 部署文档 - 杭州西湖陪拍师-宝哥人像摄影网站

## 📋 部署前准备

### 1. 替换占位内容

在部署前，请确保替换以下占位内容:

#### 图片资源
- 将 `public/placeholder/` 下的所有占位图片替换为实际作品图片
- 按照图片优化指南压缩图片
- 更新组件中的图片路径

#### 联系方式
- 编辑 `src/components/ContactModal.vue`
- 替换二维码图片
- 更新抖音账号等信息

#### 作品信息
- 编辑 `src/components/WaterfallGallery.vue`
- 编辑 `src/views/Gallery.vue`
- 更新作品标题、描述等信息

### 2. 配置网站信息

编辑 `index.html`:

```html
<title>宝哥摄影 | 杭州西湖人像摄影</title>
<meta name="description" content="杭州西湖陪拍师-宝哥，专业人像摄影" />
```

### 3. 构建生产版本

```bash
pnpm build
```

构建完成后，`dist/` 目录包含所有生产文件。

## 🚀 部署方案

### 方案一: Vercel (推荐)

#### 优点
- 免费套餐足够使用
- 自动 HTTPS
- 全球 CDN
- 一键部署
- 自动构建和部署

#### 部署步骤

1. **安装 Vercel CLI**
```bash
npm i -g vercel
```

2. **登录 Vercel**
```bash
vercel login
```

3. **部署项目**
```bash
cd /Users/nymph/projects/web/bc摄影
vercel
```

4. **配置部署参数**
```
Set up and deploy? [Y/n] Y
Which scope do you want to deploy to? (选择你的账号)
Link to existing project? [y/N] N
What's your project's name? bc-photography
In which directory is your code located? ./
Want to override the settings? [y/N] N
```

5. **生产环境部署**
```bash
vercel --prod
```

6. **访问网站**
部署完成后，Vercel 会提供一个域名，例如:
```
https://bc-photography.vercel.app
```

#### 绑定自定义域名

1. 购买域名 (如通过阿里云、腾讯云)
2. 在 Vercel 项目设置中添加域名
3. 在域名服务商添加 DNS 记录:
   - Type: `CNAME`
   - Name: `www` (或你的子域名)
   - Value: `cname.vercel-dns.com`

```
Build command: pnpm build
Publish directory: dist
```

### 方案三: GitHub Pages

#### 优点
- 完全免费
- 与 GitHub 无缝集成
- 适合开源项目

#### 部署步骤

1. **修改 Vite 配置**
编辑 `vite.config.js`:
```javascript
export default defineConfig({
  base: '/bc-photography/',  // 改为你的仓库名
  // ... 其他配置
})
```

2. **创建 GitHub 仓库**
```bash
git init
git add .
git commit -m "feat: initial commit"
git branch -M main
git remote add origin https://github.com/yourusername/bc-photography.git
git push -u origin main
```

3. **安装部署工具**
```bash
pnpm add -D gh-pages
```

4. **添加部署脚本**
编辑 `package.json`:
```json
{
  "scripts": {
    "deploy": "pnpm build && gh-pages -d dist"
  }
}
```

5. **部署到 GitHub Pages**
```bash
pnpm deploy
```

6. **启用 GitHub Pages**
- 进入仓库 Settings
- 找到 Pages 选项
- 选择 `gh-pages` 分支
- 保存并等待部署完成

7. **访问网站**
```
https://yourusername.github.io/bc-photography/
```

### 方案四: 传统服务器部署

#### 适用场景
- 有自己的 VPS 或云服务器
- 需要完全控制服务器环境
- 有特定的服务器配置需求

#### 部署步骤

1. **构建项目**
```bash
pnpm build
```

2. **上传文件到服务器**
使用 SCP 或 SFTP:
```bash
scp -r dist/* user@your-server.com:/var/www/html/
```

或使用 rsync:
```bash
rsync -avz dist/ user@your-server.com:/var/www/html/
```

3. **配置 Web 服务器**

**Nginx 配置示例** (`/etc/nginx/sites-available/bc-photography`):
```nginx
server {
    listen 80;
    server_name your-domain.com www.your-domain.com;

    root /var/www/html;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    # Gzip 压缩
    gzip on;
    gzip_types text/plain text/css application/json application/javascript text/xml application/xml application/xml+rss text/javascript;

    # 缓存静态资源
    location ~* \.(jpg|jpeg|png|gif|ico|css|js|webp)$ {
        expires 1y;
        add_header Cache-Control "public, immutable";
    }
}

# HTTPS 配置 (推荐)
server {
    listen 443 ssl http2;
    server_name your-domain.com www.your-domain.com;

    ssl_certificate /path/to/cert.pem;
    ssl_certificate_key /path/to/key.pem;

    # ... 其他配置同上
}
```

4. **启用配置**
```bash
sudo ln -s /etc/nginx/sites-available/bc-photography /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx
```

5. **申请 SSL 证书** (推荐使用 Let's Encrypt)
```bash
sudo apt install certbot python3-certbot-nginx
sudo certbot --nginx -d your-domain.com -d www.your-domain.com
```

## 🔧 域名配置

### 购买域名

推荐域名注册商:
- 阿里云万网
- 腾讯云 DNSPod
- Namecheap
- GoDaddy

### DNS 配置

#### Vercel / Netlify
添加 CNAME 记录:
```
Type: CNAME
Name: www
Value: your-app.vercel.app  (或 Netlify 提供的域名)
TTL: 600
```

#### 传统服务器
添加 A 记录:
```
Type: A
Name: @
Value: 你的服务器IP地址
TTL: 600

Type: A
Name: www
Value: 你的服务器IP地址
TTL: 600
```

## 🆘 常见问题

### 部署后页面空白

**原因**: 路由配置问题

**解决方案**:
- Vercel/Netlify: 确保配置了 SPA 重定向
- 传统服务器: 配置 Nginx/Apache 支持 HTML5 History 模式

### 图片不显示

**原因**: 图片路径错误或未上传

**解决方案**:
- 检查图片路径是否正确
- 确认图片已上传到服务器
- 查看浏览器控制台错误信息

### 样式丢失

**原因**: CSS 文件路径错误

**解决方案**:
- 检查 `vite.config.js` 中的 `base` 配置
- 确认 CSS 文件在 `dist/` 目录中
