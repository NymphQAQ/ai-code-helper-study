# AI Code Helper - 部署文档

> 版本：1.0.0 | 更新日期：2026-03-08

---

## 部署前准备

### 1. 获取 DashScope API Key

前往 [阿里云 DashScope 控制台](https://dashscope.aliyun.com/) 申请 API Key，确保账号已开通以下模型权限：

- `qwen-plus`（对话模型，含流式版本）
- `text-embedding-v3`（向量化模型，用于 RAG）

### 2. 配置检查清单

部署前确认以下几点：

- [ ] `application.yml` 中 `spring.profiles.active` 设置已确认
- [ ] API Key 通过环境变量或密钥管理系统注入，**未硬编码**
- [ ] `server.servlet.context-path` 与前端代理路径匹配
- [ ] `rag.docs-path` 在生产环境下的绝对路径已确认
- [ ] 生产 CORS 配置已收窄为具体前端域名

## 方案一：本地 / 直接运行（推荐开发调试）

### 后端启动

```bash
# 使用 Maven Wrapper（无需本地安装 Maven）
./mvnw spring-boot:run

# 使用环境变量注入 API Key（推荐）
DASHSCOPE_API_KEY=sk-xxx ./mvnw spring-boot:run

# 指定 profile
./mvnw spring-boot:run -Dspring-boot.run.profiles=local
```

后端监听：`http://localhost:8080/api`

### 前端启动

```bash
cd front
pnpm install
pnpm dev
```

前端访问：`http://localhost:5173`

## 方案二：打包 JAR 部署

### 1. 构建后端 JAR

```bash
./mvnw clean package -DskipTests
```

生成文件：`target/ai-code-helper-0.0.1-SNAPSHOT.jar`

### 2. 构建前端静态资源

```bash
cd front
pnpm build
# 输出到 front/dist/
```

### 3. 运行 JAR

```bash
java -jar target/ai-code-helper-0.0.1-SNAPSHOT.jar \
  --spring.profiles.active=prod \
  --langchain4j.community.dashscope.chat-model.api-key=sk-xxx \
  --langchain4j.community.dashscope.embedding-model.api-key=sk-xxx \
  --langchain4j.community.dashscope.streaming-chat-model.api-key=sk-xxx \
  --rag.docs-path=/data/docs
```

> 建议通过环境变量传入敏感配置而非命令行参数，防止 `ps aux` 泄露。

### 4. 前端部署

将 `front/dist/` 目录部署到 Nginx 或 CDN：

**Nginx 配置示例**：

```nginx
server {
    listen 80;
    server_name your-domain.com;

    # 前端静态资源
    root /var/www/ai-code-helper/dist;
    index index.html;

    # SPA 路由支持
    location / {
        try_files $uri $uri/ /index.html;
    }

    # 反向代理后端 API（含 SSE）
    location /api/ {
        proxy_pass http://localhost:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;

        # SSE 必需配置
        proxy_buffering off;
        proxy_cache off;
        proxy_read_timeout 3600s;
        chunked_transfer_encoding on;
    }
}
```

## 方案三：Docker 部署

### 1. 编写 Dockerfile（后端）

```dockerfile
FROM eclipse-temurin:21-jre-alpine
WORKDIR /app
COPY target/ai-code-helper-0.0.1-SNAPSHOT.jar app.jar
COPY src/main/resources/docs /app/docs
EXPOSE 8080
ENTRYPOINT ["java", "-jar", "app.jar", "--rag.docs-path=/app/docs"]
```

### 2. 构建并运行

```bash
# 构建镜像
docker build -t ai-code-helper:latest .

# 运行容器（通过环境变量注入 API Key）
docker run -d \
  -p 8080:8080 \
  -e LANGCHAIN4J_COMMUNITY_DASHSCOPE_CHAT_MODEL_API_KEY=sk-xxx \
  -e LANGCHAIN4J_COMMUNITY_DASHSCOPE_EMBEDDING_MODEL_API_KEY=sk-xxx \
  -e LANGCHAIN4J_COMMUNITY_DASHSCOPE_STREAMING_CHAT_MODEL_API_KEY=sk-xxx \
  --name ai-code-helper \
  ai-code-helper:latest
```

### 3. docker-compose.yml 示例

```yaml
version: '3.8'
services:
  backend:
    build: .
    ports:
      - "8080:8080"
    environment:
      - SPRING_PROFILES_ACTIVE=prod
      - LANGCHAIN4J_COMMUNITY_DASHSCOPE_CHAT_MODEL_API_KEY=${DASHSCOPE_API_KEY}
      - LANGCHAIN4J_COMMUNITY_DASHSCOPE_EMBEDDING_MODEL_API_KEY=${DASHSCOPE_API_KEY}
      - LANGCHAIN4J_COMMUNITY_DASHSCOPE_STREAMING_CHAT_MODEL_API_KEY=${DASHSCOPE_API_KEY}
    volumes:
      - ./docs:/app/docs   # 挂载知识库文档
    restart: unless-stopped

  frontend:
    image: nginx:alpine
    ports:
      - "80:80"
    volumes:
      - ./front/dist:/usr/share/nginx/html
      - ./nginx.conf:/etc/nginx/conf.d/default.conf
    depends_on:
      - backend
    restart: unless-stopped
```

## 方案四：云平台部署

### Vercel（前端）

```bash
cd front
pnpm build
npx vercel --prod
```

在 Vercel 项目设置中添加环境变量 `VITE_API_BASE`，并更新 `vite.config.js` 中的代理目标。

### 阿里云 ECS / 轻量服务器

1. 购买 ECS 实例（推荐 2 核 4G 以上）
2. 安装 JDK 21：
   ```bash
   apt install -y openjdk-21-jre-headless  # Ubuntu
   # 或
   yum install -y java-21-openjdk          # CentOS
   ```
3. 上传 JAR 文件并配置 systemd 服务：

```ini
# /etc/systemd/system/ai-code-helper.service
[Unit]
Description=AI Code Helper Backend
After=network.target

[Service]
Type=simple
User=www-data
WorkingDirectory=/opt/ai-code-helper
ExecStart=/usr/bin/java -jar app.jar
Environment=SPRING_PROFILES_ACTIVE=prod
Environment=LANGCHAIN4J_COMMUNITY_DASHSCOPE_CHAT_MODEL_API_KEY=sk-xxx
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
```

```bash
systemctl daemon-reload
systemctl enable ai-code-helper
systemctl start ai-code-helper
```

## 生产环境配置建议

### 安全加固

1. **API Key 管理**：使用阿里云 KMS 或 HashiCorp Vault 管理密钥
2. **CORS 收窄**：`allowedOriginPatterns` 只允许前端真实域名
3. **认证鉴权**：在 `AiController` 上添加 Token 校验（如 Spring Security JWT）
4. **HTTPS**：通过 Nginx 或云负载均衡器配置 SSL 证书

### 性能优化

1. **会话记忆持久化**：将 `PersistentChatMemoryStore` 替换为 Redis 实现，防止重启丢失历史
2. **RAG 向量库持久化**：将 `EmbeddingStore` 替换为 Milvus 或 Qdrant，避免每次启动重建索引
3. **JVM 参数调优**：
   ```bash
   java -Xms512m -Xmx2g -XX:+UseG1GC -jar app.jar
   ```

### 日志配置

```yaml
# application-prod.yml
logging:
  level:
    art.qijian.aicodehelper: INFO
    dev.langchain4j: WARN
  file:
    name: /var/log/ai-code-helper/app.log
  logback:
    rollingpolicy:
      max-file-size: 100MB
      max-history: 30
```

## 常见部署问题

### SSE 连接被 Nginx 截断

添加以下 Nginx 配置：
```nginx
proxy_buffering off;
proxy_cache off;
proxy_read_timeout 3600s;
```

### RAG 文档路径错误

生产环境使用绝对路径：
```yaml
rag:
  docs-path: /opt/ai-code-helper/docs
```

### 启动时 EmbeddingStore 初始化慢

文档较多时向量化耗时较长，建议配置异步启动或接受较长启动时间（可达 30-60 秒）。
