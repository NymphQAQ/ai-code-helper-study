# 接口文档 - 杭州西湖陪拍师-宝哥人像摄影网站

## 📋 概述

本项目为纯前端静态网站，不涉及后端 API 接口。所有数据均为静态资源，包括:
- 作品图片
- 作品元数据 (标题、描述等)
- 联系信息

以下文档为未来可能的后端扩展提供参考。

---

## 🗂️ 静态资源接口

### 1. 作品图片接口

#### 缩略图
```
GET /placeholder/thumbnail{N}.webp
```

**响应**: WebP 图片文件

**示例**:
```
/placeholder/thumbnail1.webp
/placeholder/thumbnail2.webp
```

#### 原图
```
GET /placeholder/original{N}.jpg
```

**响应**: JPEG 图片文件

**示例**:
```
/placeholder/original1.jpg
/placeholder/original2.jpg
```

## 📝 错误响应格式

所有 API 请求的错误响应遵循统一格式:

```json
{
  "success": false,
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "请求参数验证失败",
    "details": [
      {
        "field": "phone",
        "message": "手机号格式不正确"
      }
    ]
  }
}
```

### 常见错误码

| 错误码 | 说明 |
|--------|------|
| `VALIDATION_ERROR` | 请求参数验证失败 |
| `NOT_FOUND` | 资源不存在 |
| `UNAUTHORIZED` | 未授权访问 |
| `FORBIDDEN` | 禁止访问 |
| `INTERNAL_ERROR` | 服务器内部错误 |
| `RATE_LIMIT_EXCEEDED` | 请求频率超过限制 |

## 🌐 请求规范

### 基础 URL
```
开发环境: http://localhost:3000/api/v1
生产环境: https://api.yourdomain.com/api/v1
```

### 通用请求头
```
Content-Type: application/json
Authorization: Bearer <token>  (需要认证时)
```

### 响应状态码

| 状态码 | 说明 |
|--------|------|
| 200 | 请求成功 |
| 201 | 资源创建成功 |
| 400 | 请求参数错误 |
| 401 | 未授权 |
| 403 | 禁止访问 |
| 404 | 资源不存在 |
| 500 | 服务器内部错误 |

## 🔧 技术栈建议 (后端)

### 推荐方案一: Node.js + Express
- 轻量级
- 适合小型项目
- 开发效率高

### 推荐方案二: Python + FastAPI
- 高性能
- 自动生成 API 文档
- 类型安全

### 推荐方案三: Go + Gin
- 极致性能
- 编译型语言
- 适合高并发

### 数据库选择
- **MySQL**: 传统关系型数据库
- **PostgreSQL**: 功能更强大
- **MongoDB**: NoSQL，适合非结构化数据
- **SQLite**: 轻量级，适合小型项目

### 部署方案
- **Vercel/Netlify**: Serverless Functions
- **Railway**: 简单易用
- **Render**: 免费额度充足
- **Heroku**: 经典 PaaS
- **自建服务器**: 完全控制

## 📚 集成示例

### Vue 3 中使用 API

```javascript
import { ref } from 'vue'
import axios from 'axios'

const works = ref([])
const loading = ref(false)

const fetchWorks = async () => {
  try {
    loading.value = true
    const response = await axios.get('/api/v1/works')
    works.value = response.data.data
  } catch (error) {
    console.error('获取作品列表失败:', error)
  } finally {
    loading.value = false
  }
}

// 在组件中调用
onMounted(() => {
  fetchWorks()
})
```

## ⚠️ 注意事项

1. **当前状态**: 本项目为纯静态网站，暂无后端接口
2. **图片存储**: 建议使用 CDN 或对象存储 (如 AWS S3、阿里云 OSS)
3. **安全性**: 后端接口需实现身份验证和权限控制
4. **跨域**: 开发环境需配置 CORS
5. **限流**: 建议实现 API 限流防止滥用

## 🚀 快速开始 (后端)

如果需要添加后端功能，可以按以下步骤:

1. **选择技术栈** (参考上面的建议)
2. **初始化项目**
```bash
npm init -y
npm install express cors dotenv
```

3. **创建基础服务器** (`server.js`)
```javascript
const express = require('express')
const cors = require('cors')
require('dotenv').config()

const app = express()
app.use(cors())
app.use(express.json())

app.get('/api/v1/works', (req, res) => {
  res.json({
    success: true,
    data: []
  })
})

const PORT = process.env.PORT || 3000
app.listen(PORT, () => {
  console.log(`Server running on port ${PORT}`)
})
```

4. **部署后端**
- 参考部署文档中的传统服务器部署部分
- 或使用 Serverless 平台 (Vercel/Netlify Functions)
